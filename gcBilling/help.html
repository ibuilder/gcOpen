General Questions

Basic information about GCBilling application

GCBilling is a comprehensive construction billing application designed for General Contractors to manage project financial aspects including:
  • Schedule of Values management
  • Applications for Payment in AIA G702/G703 format
  • Change order tracking
  • Budget tracking
  • General Conditions tracking
  • Project team organization charts

Use the sidebar menu on the left to navigate between different sections:
  • Dashboard: Overview of your project financials
  • Project Information: Basic project details
  • Budget (GMP): Budget tracking and management
  • Applications: Manage payment applications
  • Change Orders: Track and create change orders
  • Reports: Generate various financial reports

If you encounter an error:
  1. Try refreshing the page
  2. Clear your browser cache
  3. Check your internet connection
  4. If the problem persists, contact support

The settings section allows you to:
  • Update project information
  • Manage cost codes for expense tracking
  • Configure general conditions categories
  • Set notification preferences
  • Update password and user profile

  • Application approvals

Project Information

How to manage and edit project details

To edit project information:
  1. Navigate to Settings → Project Information
  2. Update the required fields
  3. Click "Save Changes" to save your updates

Currently, GCBilling is designed to manage one project at a time. You can switch between projects using the project selector in the dashboard.

The retainage rate is the percentage of each payment that is held back until project completion. This is typically set between 5-10% and is automatically calculated in applications for payment.

  • Original Contract Sum: The initial contract amount before any changes
  • Current Contract Sum: The original amount plus approved change orders
Application for Payment

Questions about managing payment applications

To create a new application:
  1. Go to Applications section
  2. Click "New Application"
  3. Fill out the period dates and details
  4. System will auto-calculate values from SOV
  5. Review and submit

When creating an application:
  1. The system shows all SOV line items
  2. Update the % complete for each item
  3. Values are calculated automatically
  4. Review the continuation sheet for accuracy

To export in AIA format:
  1. Open the application you want to export
  2. Click "Export" button
  3. Select "AIA G702/G703" format
  4. Choose PDF or Excel format
  5. Download the file

Application statuses include:
  • Draft: In progress, not submitted
  • Pending: Submitted, awaiting approval
  • Approved: Approved by client
  • Rejected: Requires revisions
  • Paid: Payment received
Schedule of Values

Information about managing SOV line items

A Schedule of Values is a detailed breakdown of the contract sum into various work items or components. It's used to track progress and calculate payment applications throughout the project.

To create SOV items:
  1. Go to Schedule of Values section
  2. Click "Add Item"
  3. Enter item number, description, and value
  4. Assign appropriate cost code
  5. Save the item

Best practices for organizing SOV:
  • Use CSI MasterFormat divisions for numbering
  • Group similar work items together
  • Include general conditions as line items
  • Break down major components into manageable items
  • Ensure the total equals the contract sum

Yes, but with limitations:
  • You can add new items for change orders
  • Descriptions can be updated
  • Values should not be changed after approved applications
  • Use change orders to modify contract values
General Conditions

Managing general conditions and categories

General Conditions are indirect project costs that support the construction work but aren't part of the physical building. Examples include:
  • Project management staff
  • Temporary facilities
  • Site security
  • Equipment rental
  • Quality control testing

To add GC categories:
  1. Go to Settings → General Conditions
  2. Click "Add Category"
  3. Enter category name and description
  4. Save the category
  5. These categories help organize GC items

Track GC expenses by:
  1. Creating GC line items in your SOV
  2. Assigning appropriate categories
  3. Updating progress in applications
  4. Monitoring actual vs budgeted costs

Yes, General Conditions should be included in the SOV as line items. This allows you to:
  • Bill for GC costs progressively
  • Track actual vs planned GC expenses
  • Maintain transparency with the client
  • Include GC in payment applications
